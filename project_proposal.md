### Project Description

We intend to build a food comparison application that will let users search for or scan an item and then compare it to other items, and search for healthier alternatives. We’ll use the **OpenFoodFacts API** for food and nutrition information. Since OpenFoodFacts is an open-source API, we’ll need to handle messy or incomplete data by normalising, cleaning, structuring, and validating the information before using it in our mobile application.  

For camera scanning, we’ll use **react-native-camera** and **expo-barcode-scanner** to scan barcodes.

We plan to use **React Native** to build out the frontend of our app since it will be on mobile platforms. This will allow us to learn a new skill — mobile development — while leveraging our existing React experience from the Full-Stack Development module. For the backend, we’ll use **Django**, which we’re already familiar with from Full Stack.  

We’ll connect a **PostgreSQL** database to gain more experience with databases in applications. This requires only minor configuration changes in `settings.py`.  

We’ll also experiment with integrating **Large Language Model (LLM)** APIs to generate meaningful insights from food data. By feeding nutritional values into these models, we’ll develop our **prompt engineering** skills and learn how to extract useful, context-aware results from AI systems.

By building this project from scratch, we’ll gain hands-on experience with the **full software development lifecycle** — from planning and implementation to testing and deployment — while becoming more confident in **DevOps** practices and using **Git** for collaboration.
